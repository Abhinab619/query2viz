Core Stack
🤖
LangChain
Agent orchestration framework. Used create_tool_calling_agent, AgentExecutor, ChatPromptTemplate, MessagesPlaceholder, and StructuredTool.
Orchestration
⚡
Groq + LLaMA 3.1
Inference backend via langchain_groq.ChatGroq. Model: llama-3.1-8b-instant. Chosen for its 128k context window and extremely fast inference speed.
LLM Inference
🗄️
SQLite3
Local database accessed via Python's built-in sqlite3 module. Schema introspected at runtime using PRAGMA table_info and PRAGMA foreign_key_list.
Database
📊
Matplotlib
Graph rendering engine. Supports bar, line, scatter, and pie charts. Saves output as .png files with tight layout and rotated x-tick labels.
Visualization
🧩
Pydantic
Input schema validation for tools. QueryInput validates SQL strings; GraphInput enforces List[str] labels, List[float] values, and graph_type.
Validation
🔑
python-dotenv
Loads GROQ_API_KEY from a .env file via load_dotenv(). Keeps secrets out of source code.
Config
Custom-Built Components
🔧
describe_tables Tool
Built with @tool decorator. Dynamically discovers full DB schema — all tables, column types, primary keys, foreign keys, and relationship strings — and returns it as a Python dict to the LLM.
@tool decorator PRAGMA queries
▶️
run_query_tool
Built with StructuredTool.from_function(). Accepts a SQL string via QueryInput Pydantic schema, runs it against SQLite, returns rows or a formatted SQL error string.
StructuredTool Error handling
📈
graph_tool
Built with StructuredTool.from_function(). Accepts filename, labels, values, and graph_type. Renders via matplotlib, saves as .png, and updates shared state.
matplotlib State update
📡
ToolCaptureHandler
Custom BaseCallbackHandler. Intercepts on_tool_end() events and stores the last tool's output in last_tool_output using ast.literal_eval() to parse string-wrapped Python objects.
Callback system Inter-agent data
